# Security Policy

This repository hosts a static website (no server-side code, no user data, no
authentication). The security surface is limited to the published static assets.

## Reporting a vulnerability

If you find a security issue (for example, a malicious or compromised link, or
an exposed secret in the repository), please report it privately:

1. Preferred: open a private security advisory via GitHub
   (Security tab, "Report a vulnerability") on this repository.
2. Or email **support@jgsystemsconsulting.com**.

Please do not open a public issue for a security report. We aim to acknowledge
reports within a few business days.

Vulnerabilities in the individual products linked from this site should be
reported against their own repositories, each of which carries its own
`SECURITY.md`.
